## Conventions Established
- **Mount sync watcher must suppress events where the tracked hash is unchanged**: Remote pulls write atomically and trigger fsnotify; echoing those back as pushes advances remote revisions and causes multi-mount conflicts. (scope: internal/mountsync and any future local-watcher → remote-push path)
- **Prefer the export snapshot endpoint for bulk/initial reads; keep tree+read as a fallback**: One request vs N+1 requests; older servers may still lack export. (scope: CLI mount bootstrap and any sandbox/agent tools that need a workspace snapshot)
- **Pass runtime CLI config to the hosted observer via URL fragment, never query string**: Fragments stay client-side, avoiding credential exposure to the observer host and intermediaries; works without per-user NEXT_PUBLIC build values. (scope: Any CLI→hosted-web launcher that needs to inject server/workspace/token)
- **When assembling URLs in Go, single-encode the fragment string and assign to url.URL.Fragment directly**: url.URL.String() re-encodes Fragment; calling Encode() first produces double-encoded output. (scope: All Go URL construction where a fragment carries encoded params)
- **Workspace resolution order: explicit flag > RELAYFILE_WORKSPACE > token claim > stored default; list fallback surfaces active workspace first**: Agent/scoped tokens often lack admin scopes for remote workspace listing. (scope: relayfile CLI commands and docs)
- **npm-published Go CLIs must pack platform binaries and chmod them after artifact download in the publish workflow**: GitHub artifacts strip exec bits; users commonly install with --ignore-scripts. (scope: Any @relayfile/* or sibling package that ships a Go binary via npm)
- **Writeback/edit paths must use optimistic concurrency (If-Match) and fs:write-scoped tokens, and refresh details after save**: Relayfile writes require the revision check and the scoped token; UI must reflect the new state immediately. (scope: File observer edit UI and any other Relayfile write clients)
- **Workers check `trail status` before `trail start` and reuse inherited workflow trajectories in automated runs**: Prevents duplicate trajectories when the orchestrator has already opened one for the workflow step. (scope: AGENTS.md / workflow worker prompts)
- **DB-state-dependent features gate merges on PGlite-backed integration tests, not mocks**: Per the 80-to-100 skill; workspace_integrations mocks were insufficient evidence for the writeback bridge. (scope: Workflows that exercise cloud DB paths (writeback, provider sync, etc.))
- **Observer panels use viewport-bounded independent scroll regions; file details are sticky on xl with internal overflow**: Page-level sticky fails when ancestors bound overflow or at narrower breakpoints; independent scroll regions keep details visible across layouts. (scope: packages/file-observer dashboard layout)
- **Derive provider filter options from root path segments (/github, /notion, /slack) rather than relying on provider metadata**: Root provider directories don't always carry provider metadata, so the filter must work before child files load. (scope: File observer filter/selector components)

## Lessons Learned
- Local file watchers need explicit echo-suppression when the same process also writes files from remote pulls (The mount sync watcher was re-pushing atomically-written remote pulls, advancing revisions and breaking e2e Agent B edit propagation.) - Gate pushes on a content-hash diff against the last-known tracked hash; add a regression test that pulls a remote edit and asserts no outbound push event.
- Verification signals must check artifact state, not just stdout (design-dashboard retried because the verifier read the agent's stdout instead of the already-completed DESIGN.md file ending with DESIGN_DONE.) - Have workflow verifiers read the target artifact on disk (or channel-posted evidence) as the primary signal; treat stdout as secondary.
- Go's url.URL fragment re-encoding is a silent double-encode trap (Assigning fragment.Encode() to URL.Fragment produced browser-side parameters that failed to decode.) - Build the fragment string once with the intended single encoding and assign directly; add a test that parses URL.String() as a browser would.
- Pipelines that `| tail` or broadly grep can mask real failures and create false-positive safety gates (PR 46/writeback-workflow review found npm install failures hidden by a tail pipe and a secret-grep flagging legitimate NANGO_SECRET_KEY refs.) - Preserve the root-cause exit code (PIPESTATUS / set -o pipefail), and anchor secret patterns to avoid matching documented env var names.
- npm packages that install Go binaries must assume --ignore-scripts (Users install with scripts disabled; postinstall-only binary fetches leave `bin/relayfile` missing. Artifact downloads also strip Unix mode bits.) - Pack platform binaries into the tarball, chmod +x them after artifact download in the publish workflow, and have bin/relayfile dispatch to the packed binary.
- Defaults should match the actual hosted surface, not aspirational hostnames (Both observer (files.relayfile.dev) and CLI server defaults were out of sync with the live api.relayfile.dev / agentrelay.com routes.) - Treat default hostnames as a deploy-coupled config and add a CLI/doc regression test any time the hosted surface moves.
- Scoped tokens break any UI that assumes admin scopes (Workspace listing required admin scopes; scoped agent tokens hit an empty/forbidden list with no clear workspace context.) - Always derive the active context from env/token claims as a fallback and surface it ahead of remote listings.
- Page-level sticky positioning is fragile across breakpoints and ancestor overflow (File details pane disappeared on non-xl widths even after adding `sticky` because ancestor overflow bounded the sticky area.) - Use independent viewport-bounded scroll regions (flex + min-h-0 + overflow-auto) rather than page-level sticky for persistent side panels.
